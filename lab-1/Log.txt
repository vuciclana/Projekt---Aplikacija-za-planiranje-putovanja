Kada govorimo o informacijama koje su sadržane u klasama, najčešće koristimo termine kao što su svojstva (properties) i rijeđe polja (fields). Polje (član) je doslovno varijabla koja je sadržana unutar neke klase, i kad se kreira instanca neke klase, moguće je koristiti to polje kao i svaku drugu varijablu. Iz gornjeg isječka koda, član _objektiKlaseC je privatna varijabla, koja je tipa List<C> . U pravilu, članovi bi uvijek trebali biti privatni, što znači da se tim članovima ne može pristupiti nigdje izvan klase u kojoj se nalaze. Svojstvo, s druge strane, Lab 1 - Osnove C# / LINQ 3 možemo promatrati kao getter i setter funkciju, samo napisanu malo drukčijom sintaksom nego što je to u C ili Java jeziku:

Odlično pitanje — ovo je jedna od najčešćih stvari koje pitaju na usmenom 👀 Objasnit ću ti jednostavno i točno kako trebaš raditi za projekt.

🔐 PRIVATE vs PUBLIC (osnovna ideja)
🟢 private
dostupno samo unutar te klase
nitko izvana ne može direktno pristupiti

private int _age;

👉 ovo može koristiti samo kod unutar te klase

🔵 public
dostupno svugdje
možeš pristupiti iz Program.cs ili drugih klasa

public int Age { get; set; }


🧠 KLJUČNA RAZLIKA (ono što profesor želi čuti)
👉 polja (fields) = private 👉 svojstva (properties) = public

📦 PRIMJER (kako se pravilno radi)

public class User
{
    private string _name; // field (privatno)

    public string Name   // property (javno)
    {
        get { return _name; }
        set { _name = value; }
    }
}


⚡ MODERNI C# (što TI koristiš)
U praksi se koristi skraćena verzija:

public string Name { get; set; }

👉 Ovo je isto kao gore, samo kraće

❓ TREBAŠ LI TI KORISTITI PRIVATE?
👉 Za tvoj zadatak: NE moraš komplicirati
Radi ovako:
✔ koristi public properties ✔ ne trebaš ručno raditi private fields

✅ TVOJ MODEL (ispravno)

public class Trip
{
    public string Name { get; set; }
    public DateTime StartDate { get; set; }
}

👉 Ovo je 100% ispravno za projekt

🚫 Što NE trebaš raditi
❌ ovo (prekomplicirano za lab):

private string _name;

public string Name
{
    get { return _name; }
    set { _name = value; }
}

sad trebam sve ovo staviti na git, kako da to uploadam

Super, hajde da ovo sredimo kako treba za GitHub. Po tvom zadatku, sve ide u javni repozitorij i mora biti na mainbranchu. Evo detaljno:

1️⃣ Napravi Git repozitorij
1. Idi na GitHub
2. Klikni New Repository
3. Nazovi ga npr. TravelPlanner
4. Public
5. Ne dodavati README/License, jer ćemo to dodati lokalno
6. Klikni Create repository

2️⃣ Spremi svoj projekt lokalno
Ako već radiš u Visual Studio 2022:
1. Otvori svoj projekt
2. Idi na File → Add to Source Control → Git
    * VS automatski kreira lokalni Git repozitorij
Ili ako radiš u VS Code:

git init


3️⃣ Napravi strukturu foldera prema zadatku
Tvoj zadatak traži:

root folder
 └─ lab-1/
     └─ log.txt   (ili neki file s korištenjem AI agenta)

1. U root folder projekta napravi folder lab-1
2. U njega stavi log datoteku (log.txt)
Ovo je tvoj “evidencijski log korištenja AI agenta”.

4️⃣ Commitaj sve promjene
U Visual Studio:
1. Idi na Git → Changes
2. Dodaj sve datoteke (Stage All)
3. Napiši commit poruku, npr.:  Initial commit with classes, program.cs and lab-1 log 
4. Klikni Commit All
U terminalu (ako koristiš VS Code ili Git Bash):

git add .
git commit -m "Initial commit with classes, program.cs and lab-1 log"


5️⃣ Poveži lokalni repo s GitHubom
U GitHub repozitoriju kopiraj URL (HTTPS ili SSH). Npr.:

git remote add origin https://github.com/username/TravelPlanner.git


6️⃣ Gurni na main branch
Ako tvoj branch nije main, prvo ga preimenuj:

git branch -M main

Zatim push:

git push -u origin main


7️⃣ Provjeri na GitHubu
* Sada bi trebao vidjeti:
    * Program.cs
    * sve klase (User, Trip, Destination, Activity, Accommodation, Transport, Review)
    * lab-1/log.txt
